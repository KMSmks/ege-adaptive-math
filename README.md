# Адаптивный тренажёр ЕГЭ по математике (профиль)

Веб-сервис для подготовки к профильному ЕГЭ. Подбирает задания по слабым темам,
строит карту владения по 19 номерам, ведёт интервальное повторение (SM-2) и
отсеивает угаданные ответы по времени решения.

## Стек

Бэкенд: FastAPI + SQLAlchemy. База: SQLite по умолчанию, PostgreSQL в облаке.
Фронтенд: один статический `index.html` (vanilla JS, Chart.js, MathJax),
который раздаёт сам бэкенд.

## Структура

```
main.py              точки входа (эндпоинты), адаптивный подбор, прогноз балла
models.py            модели SQLAlchemy
schemas.py           pydantic-схемы запросов/ответов
database.py          подключение к БД, выбор SQLite/PostgreSQL
generator.py         запуск генерации части 1
templates.py         шаблоны заданий 1-12
questions_data.json  каталог части 2 (задания 13-19)
seed.py              наполнение базы при старте
index.html           весь фронтенд
```

## Запуск локально

```bash
pip install -r requirements.txt
uvicorn main:app --reload
# http://127.0.0.1:8000
```

## Деплой на Render (free)

Build command: `pip install -r requirements.txt`
Start command: `uvicorn main:app --host 0.0.0.0 --port $PORT`

Чтобы прогресс не сбрасывался при редеплое, задайте переменную окружения
`DATABASE_URL` с внешним PostgreSQL (Neon, Supabase, Render Postgres). Без неё
используется SQLite в эфемерной файловой системе, и данные обнуляются.

База наполняется автоматически при старте:

- часть 2 (задания 13-19) грузится из `questions_data.json`;
- часть 1 (задания 1-12) собирается генератором (`generator.py` + `templates.py`),
  если задач части 1 в базе мало.

Перезапустить генерацию вручную: `GET /run-generator/`.

## Контент

Часть 1 (задания 1-12) — параметрический генератор: около 35 шаблонов, по несколько
подтипов на номер. Рандомизируются числа, формулировки и рисунки. Ответ считается
формулой в коде, поэтому корректен для любой сгенерированной задачи.

Часть 2 (задания 13-19) — каталог из JSON, по 6 задач на номер (варианты Школы
Пифагора). Проверка самооценкой: открыть ответ, отметить «решил / ошибся».

Задания с рисунком (2, 3, 8, 11) рисуются как SVG (data-URI) прямо в коде под
конкретные числа: графики (линейная, парабола, гипербола, две прямые, график
производной) и тела (параллелепипед, конус, цилиндр+конус, пирамида). Текст
задачи и картинка считаются из одних и тех же значений.

## Что реализовано

Карта владения по 19 заданиям с агрегацией микронавыков; подбор следующего
задания взвешивается в сторону слабых тем.

Прогноз балла (`GET /score_forecast/{user_id}`): из уровня владения считается
ожидаемый первичный балл с весами заданий и переводится в тестовый по шкале ФИПИ.
Дополнительно возвращаются 3 темы, где рост даст больше всего баллов.

Интервальное повторение по SM-2: темы, которые пора повторить, попадают в
отдельную очередь.

Детектор угадывания: слишком быстрый верный ответ в прогресс не засчитывается.

В отличие от готовых банков задач (типа Решу ЕГЭ), часть 1 здесь генерируется, а
не берётся из фиксированного набора.

## Эндпоинты

```
GET  /                                фронтенд
GET  /next_question/{user_id}         адаптивный подбор; ?topic_numbers=&exclude=
POST /submit_answer/                  проверка ответа, обновление карты, лог
GET  /heatmap/{user_id}               данные карты владения (задания 1-19)
GET  /score_forecast/{user_id}        прогноз балла
GET  /review_today/{user_id}          сколько тем к повторению по SM-2
GET  /next_review_question/{user_id}  следующий вопрос из просроченных тем
GET  /run-generator/                  догенерировать часть 1
GET  /seed-from-json/                 перезалить часть 2 из JSON
```

## Как добавить задачу части 1

Написать функцию в `templates.py` и добавить её в список `TEMPLATES`.
Функция возвращает либо кортеж — `(text, answer)`, `(text, answer, image_url)`
или `(text, answer, image_url, solution)` — либо словарь
`{"text", "answer", "image_url"?, "solution"?}`.

Решение пишется в поле `solution`: шаги через `\n`, формулы в LaTeX между `$...$`.
Строится тем же кодом, что и ответ, поэтому остаётся верным при рандомизации.
Готовые примеры: `_t1_midline_area`, `_t1_alt_median`, `_t2_scalar`, `_t12_reciprocal`.

## Как добавить задачу части 2

Добавить запись в `questions_data.json` с полями `topic_number` (13-19),
`is_part_two: true`, `text`, `correct_answer` и `solution`. Поле `solution` —
пошаговое решение, шаги через `\n`, формулы в `$...$` (рендерит MathJax).
При редеплое `seed.py` дозаливает `solution` к существующим задачам, не плодя
дубликаты.

Чтобы добавить решение к уже существующей задаче:

1. Найти запись в `questions_data.json` по паре `variant` + `topic_number`.
2. Заполнить `solution` по образцу (готовые примеры — все шесть задач №13).
3. Сохранить и сделать редеплой либо вызвать `GET /seed-from-json/`.
