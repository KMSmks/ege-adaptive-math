# Адаптивный тренажёр ЕГЭ по математике (профиль)

Веб-сервис для подготовки к ЕГЭ: адаптивный подбор заданий по слабым темам,
тепловая карта владения, интервальная тренировка, детектор угадывания.

## Стек
FastAPI + SQLAlchemy, SQLite по умолчанию / PostgreSQL в облаке.
Фронтенд — один статический `index.html` (vanilla JS + Chart.js), раздаётся бэкендом.

## Запуск локально
```bash
pip install -r requirements.txt
uvicorn main:app --reload
# открыть http://127.0.0.1:8000
```

## Деплой на Render (free)
- Build command: `pip install -r requirements.txt`
- Start command: `uvicorn main:app --host 0.0.0.0 --port $PORT`
- Для сохранения прогресса между перезапусками задайте переменную окружения
  `DATABASE_URL` (внешний PostgreSQL: Neon / Supabase / Render Postgres).
  Без неё используется SQLite в эфемерной ФС — данные обнулятся при редеплое.

База наполняется **автоматически при старте**:
- часть 2 (задания 13–19) — каталог из `questions_data.json`;
- часть 1 (задания 1–12) — параметрический генератор (`generator.py` + `templates.py`),
  запускается, если задач части 1 в базе мало.
Ручной перезапуск генерации: открыть `…/run-generator/`.

## Архитектура контента
| Часть | Задания | Источник | Вариативность |
|------|---------|----------|---------------|
| 1 | 1–12 | генератор шаблонов | бесконечная (значения, текст и SVG-рисунки считаются в коде) |
| 2 | 13–19 | каталог `questions_data.json` | 6 задач на номер (варианты Школы Пифагора) |

Ответы части 1 вычисляются формулами — всегда корректны. Задания с рисунком
(2, 3, 8, 11) рисуются как SVG (data-URI) прямо в коде, текст и график согласованы.
Часть 2 проверяется самооценкой (показать ответ → «решил верно / ошибся»).

## Как добавить
- **Шаблон части 1:** функция в `templates.py`, возвращает `(text, answer)` или
  `(text, answer, image_url)`; добавить строку в `TEMPLATES`.
- **Задача части 2:** запись в `questions_data.json`
  (`topic_number` 13–19, `is_part_two: true`, `text`, `correct_answer`).

## Основные эндпоинты
- `GET /` — фронтенд
- `GET /next_question/{user_id}?topic_numbers=...&exclude=...` — адаптивный подбор
- `POST /submit_answer/` — проверка ответа, обновление тепловой карты, лог
- `GET /heatmap/{user_id}` — данные тепловой карты (по заданиям 1–19)
- `GET /run-generator/` — догенерировать часть 1
- `GET /seed-from-json/` — перезагрузить часть 2 из JSON
